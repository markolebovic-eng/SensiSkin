When routing a "what should we blog about / trending topics" task, this is a single-domain SEO task — do NOT spawn parallel agents. Brief the SEO specialist to do all of: (1) live demand validation (Google Trends + google_suggest + PAA), (2) direct competitor blog audits to find coverage GAPS not just topics, (3) international trend-lag analysis (trends reaching Serbia/Balkans typically lag global by 3-6 months = "get there first" windows), (4) map to the client's actual service list.

**Why:** A topic recommendation is only actionable if it (a) has demand, (b) has a competitor gap, and (c) won't cannibalize the client's existing ranked pages. The trend-lag angle repeatedly surfaces high-value, low-competition topics (e.g. "skin longevity" for SensiSkin in mid-2026 — dominant term, zero Novi Sad competitor coverage yet).

**How to apply:** Always include the client's existing keyword table + published-post list in the brief and demand an explicit collision check per recommendation. Blog keywords target INFORMATIONAL intent and must be a distinct phrase from service-page transactional keywords even when the same service is the target (e.g. blog "LED terapija svetlom za kožu" vs service page "LED terapija za kožu lica"). Require topic titles + keywords in the client language, report body in English.
